# AGENTS.md — контекст проекта IPbrowse

Этот файл — рабочий контекст для AI-ассистентов и разработчиков. Здесь
описано, как устроен проект, как его запускать / собирать / проверять,
какие правила стиля приняты и какие важные решения принимались по ходу
разработки. **Обновляйте этот файл при каждом значимом изменении.**

## Что это за проект

IPbrowse — настольный сканер локальной сети с GUI на PySide6
(Qt 6). Тёмная тема Catppuccin. Интерфейс полностью на русском
языке.

Возможности (актуальный набор):

- Ping-сканирование подсетей / диапазонов / списков IP.
- Резолв имён (reverse DNS), MAC из ARP-таблицы, производитель по OUI.
- Сканирование TCP-портов (TCP-connect, без админ-прав).
- nmap-подобные флаги: `-Pn`, `-PR`, `-n`, `--no-mac`, `-O`, `-sV`,
  `-T0..T5`, `--host-timeout`, `-sn`, `-F`, `-p-`, `--top-ports N`,
  `--randomize-ports`, `--randomize-hosts`, `--max-parallel`,
  `--retry`, `--exclude`, `-oN/-oX/-oG`.
- Снятие баннеров (`-sV`) и грубое определение ОС по TTL (`-O`).
- Wi-Fi: текущая сеть, шлюз, роутер.
- Массовое сканирование одного порта по списку IP.
- Экспорт результатов: CSV / TSV / TXT, выбор полей и пресетов.
- Per-host диалог открытых портов с расшифровкой сервиса (IANA),
  типичным софтом и кнопкой **«Узнать больше»** — открывает поиск
  Google на русском языке (`hl=ru`, запрос вида «что такое порт N tcp»).
- Вкладка «О программе».
- **Авто-установка Nmap.** Если на старте `shutil.which("nmap")`
  возвращает `None`, открывается модальный `NmapInstallDialog`. Под
  Windows он скачивает официальный setup.exe (последняя версия
  парсится с `nmap.org/download.html`, fallback на `7.95`) и
  запускает его — UAC поднимается автоматически. Под Linux / macOS
  определяется пакетный менеджер (`apt` / `dnf` / `pacman` /
  `brew` / …), системный терминал открывается с подставленной
  командой `sudo … install nmap`. Чекбокс «Больше не показывать»
  сохраняется в `QSettings("IPbrowse", "IPbrowse")`,
  ключ `nmap/dont_ask_again`.

## Структура

```
IPbrowse/
├── app.py            # PySide6 GUI (MainWindow, вкладки, диалоги)
├── scanner.py        # сетевая логика (ping, ARP, TCP-сканер, OUI)
├── _smoke.py         # офлайн smoke-тест экспорт-диалога mass-scan
├── ports.csv         # реестр портов IANA, грузится в PORT_SERVICE
├── resources/
│   └── icon.png      # иконка приложения (используется PyInstaller'ом)
├── IPbrowse.spec     # spec для PyInstaller
├── requirements.txt
├── README.md
└── AGENTS.md         # этот файл
```

## Зависимости

Минимум Python 3.9. Зависимости перечислены в `requirements.txt`:

```
PySide6>=6.5.0
mac-vendor-lookup>=0.1.12
```

`mac-vendor-lookup` при первом запуске подкачивает базу OUI в
~/.cache.

## Как запустить

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows (cmd / PowerShell)
# source .venv/bin/activate     # Linux / macOS
pip install -r requirements.txt
python app.py
```

Запуск без GUI (только сканер):

```bash
python scanner.py 192.168.1.0/24
```

## Как проверить (verification)

Перед коммитом запускайте обе команды:

```bash
python -m py_compile app.py scanner.py _smoke.py   # синтаксис
python _smoke.py                                   # smoke-тест
```

Smoke-тест офлайн, не требует сети, не открывает окон. Завершается
строкой `ALL OK` при успехе. Падение / трассировка → коммит не
делать, чинить.

Опционально, ручная проверка GUI:

```bash
python app.py
```

— открыть вкладки, нажать «Сканировать», выбрать строку → пункт
«Порты…», нажать «Узнать больше» → должна открыться вкладка
google.com на русском языке.

## Сборка бинарника

Бинарники собираются из CI на push тэга (см.
`.github/workflows/`, ветка `main` фиксирует ход разработки).
Локально:

```bash
pyinstaller IPbrowse.spec
# → dist/IPbrowse.exe (Windows), dist/IPbrowse (Linux / macOS)
```

Spec-файл подключает `ports.csv` через `datas=[('ports.csv', '.')]`
и иконку `resources/icon.png` через параметр `icon=`. Файлы
ресурсов больше пока не бандлятся.

## Стиль кода и важные соглашения

- **UI-тексты только на русском.** Все строки видимые пользователю
  должны быть на русском. Английский допустим только в комментариях
  кода, docstring'ах и идентификаторах.
- **Никаких эмоджи / пиктограмм в GUI.** Уже была отдельная зачистка
  (коммит `c7ffe28`), не возвращать.
- **Тёмная тема Catppuccin.** Палитра в `_apply_dark_theme`, акценты
  `#89b4fa` (синий), `#a6e3a1` (зелёный), фоны `#1e1e2e` / `#181825`
  / `#313244` / `#45475a`.
- **Стиль чекбоксов:** галочка вместо заливки. Иконка генерируется
  один раз в `tempfile.gettempdir()` функцией
  `_checkmark_icon_path()` и подставляется в QSS-плейсхолдер
  `__CHECK_PATH__` через `str.replace`. **Не ломать:** placeholder
  выбран, чтобы избежать массового экранирования `{}` в QSS, который
  иначе пришлось бы делать под f-strings.
- **Сканер — TCP-connect.** SYN/UDP/IP-protocol/`-sC`/decoy/etc
  невозможны без nmap или прав администратора, диалог флагов
  явно об этом сообщает.
- **Nmap опционален.** IPbrowse работает и без него — но если nmap
  не найден, при старте появляется окно с предложением установить.
  Не убирать диалог из `MainWindow.__init__`: пользователь явно
  попросил эту фичу. Версия и URL стоят на одной странице
  `nmap.org/download.html`; если поломался парсер версии —
  сначала проверьте, не сменился ли формат имени файла
  (`nmap-X.YZ-setup.exe`).
- **`flat-list` экспорт.** Колонки в CSV / TSV выбираются через
  `MassExportDialog`, заголовок зависит от `with_header`. Тестовое
  покрытие — `_smoke.py`.
- **Не дёргать сеть в тестах.** `_smoke.py` офлайн, не открывает
  окон, использует patch на `MassExportDialog.exec` /
  `QFileDialog.getSaveFileName` / `QMessageBox.information`. Если
  добавляете новые тесты — придерживайтесь того же подхода.

## Git workflow

- Ветка `main` — основная.
- Сообщения коммитов короткие и предметные (см. `git log`):
  `Mass scan: only show opens, fix freeze, auto-scroll new rows`,
  `Add 'О программе' tab + per-host ports drill-down`, и т.п.
- Релизы — теги `vX.Y.Z`, на пуш тэга срабатывает CI и собирает
  бинарники на Linux / Windows / macOS.
- **Коммитим и пушим после каждого осмысленного шага** — это явное
  пожелание владельца репозитория.
